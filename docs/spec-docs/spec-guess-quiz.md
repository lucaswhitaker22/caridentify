# Spec guess quiz

### Purpose

Add a short skill-based moment after identification. Turn car knowledge into bonus engagement.

### Product goals

* Reinforce learning.
* Add variety to successful scans.
* Create optional bonus rewards.

### Core concept

After a successful identification, the app can ask one quick car spec question. A correct answer grants a bonus.

### Quiz rules

* The quiz is optional.
* One question appears per eligible result.
* Questions should be short and easy to answer fast.
* Answers resolve immediately.

### Question types

* Horsepower.
* Drivetrain.
* Engine type.
* Year range.

### Reward principles

* Correct answers grant a small bonus.
* Incorrect answers should not punish core progression.
* The bonus should feel nice, not essential.

### Learning principles

* Questions should teach through feedback.
* Correct answers should reinforce vehicle identity.
* Repeated exposure should improve player familiarity.

### UX principles

* Present the quiz after the main result, not before.
* Keep the question readable in one glance.
* Let players skip without friction.

### Guardrails

* Do not block the main loop behind quiz participation.
* Do not make answers too obscure.
* Do not overuse quizzes in every session.

### Success criteria

* Players engage with quizzes voluntarily.
* The feature adds variety without slowing the loop.
* Players retain more vehicle knowledge over time.

### Out of scope

* Question generation systems.
* Technical answer validation.
