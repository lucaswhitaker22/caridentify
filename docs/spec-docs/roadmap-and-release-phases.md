# Roadmap and release phases

### Purpose

Align the team on phased delivery. Define what success looks like at each stage.

### Product goals

* Validate the core loop early.
* Add retention systems in controlled phases.
* Expand only after the foundation works.

### Phase 0 — Prototype

#### Goal

Prove that the capture and identification loop is compelling.

#### Scope

* Basic camera flow.
* Identification result.
* Garage entry creation.
* Minimal reward feedback.

#### Exit criteria

* Players complete the loop quickly.
* First sessions feel intuitive.
* Core concept tests well with target users.

### Phase 1 — MVP

#### Goal

Launch the first replayable version.

#### Scope

* Rarity system.
* Daily spotlight challenges.
* XP and coins.
* Leaderboards.
* Basic fair play controls.

#### Exit criteria

* Players return for repeated sessions.
* Core reward loops feel clear.
* Early abuse is manageable.

### Phase 2 — Social and collection depth

#### Goal

Add social stickiness and long-term goals.

#### Scope

* Trading.
* Achievement badges.
* Collections and sets.
* Optional bonus loops.

#### Exit criteria

* Social features drive repeat use.
* Collection depth increases session value.
* More players pursue long-term goals.

### Phase 3 — Live ops and revenue

#### Goal

Keep the product fresh and sustainable.

#### Scope

* Seasonal content.
* Cosmetic monetization.
* Garage expansion.
* Ongoing event planning.

#### Exit criteria

* Events improve retention.
* Monetization remains fair.
* Content cadence feels sustainable.

### Planning guardrails

* Do not overload the MVP.
* Validate one major system before adding the next.
* Keep feature sequencing tied to player behavior.

### Out of scope

* Team resourcing plans.
* Technical delivery schedules.
