# Progression, XP, and coins

### Purpose

Define how progress feels, accumulates, and stays understandable. Make every session move the player forward.

### Product goals

* Reward both first discoveries and repeat play.
* Give players short-term and long-term goals.
* Keep the economy easy to understand.
* Prevent any single activity from dominating value.

### Player value

Progression answers one core question. "Why should I play one more round?"

XP gives visible account growth. Coins give flexible spending power. Together they turn captures into lasting progress.

### System role in the product

This system sits between the core loop and long-term retention.

* The core loop creates rewards.
* Progression turns rewards into momentum.
* Other systems spend or amplify that momentum.

### Core currencies

#### XP

XP measures account progress. It increases player level and marks overall experience.

XP should mainly communicate advancement. It should not feel like a spendable resource.

#### Coins

Coins are the flexible reward currency. They support upgrades, access, convenience, and cosmetic goals.

Coins should feel useful often. They should not feel scarce in normal play.

### Why two currencies exist

XP answers status and unlock pacing. Coins answer choice and utility.

This split keeps progression readable. Players know what they earned and why it matters.

### Reward principles

* New car discoveries grant the highest base reward.
* Rare cars grant higher rewards than common cars.
* Daily participation gives consistent value.
* Bonus rewards should feel meaningful, not mandatory.
* Small sessions should still create visible progress.
* Duplicates should stay useful, but less valuable than discoveries.
* Premium purchases must not replace core reward loops.

### Reward sources

#### Core play rewards

* Successful captures.
* First-time discoveries.
* Duplicate conversions.

These rewards support the main habit. Players should earn something from almost every valid session.

#### Goal-driven rewards

* Challenge completion.
* Collection completion.
* Achievement milestones.
* Quiz bonuses.

These rewards create spikes. They should feel memorable and directional.

#### Return-play rewards

* Streak rewards.
* Daily spotlight completion.
* Seasonal event participation.

These rewards support retention. They should improve return behavior without feeling mandatory.

### Reward weighting guidance

Use reward weight to teach player priorities.

* First discoveries should pay the most.
* Rare discoveries should create strong spikes.
* Duplicates should pay enough to avoid disappointment.
* Daily systems should supplement, not replace, exploration.

### Leveling model

* Levels should unlock status and access.
* Early levels should come quickly.
* Mid-game levels should slow at a predictable pace.
* Leveling should never reset collection progress.

#### Early game

Early levels should teach the economy fast. Players should see multiple level-ups in the first sessions.

This phase proves momentum. It should build confidence before complexity appears.

#### Mid-game

Mid-game should shift from novelty to intentional goals. Progress should stay steady, but feel earned.

Players should start chasing rarity, sets, and dailies here. Leveling should support that shift.

#### Late game

Late-game levels should represent dedication. They should signal prestige, not block enjoyment.

Progress should remain visible even when level-ups slow. Side goals become more important here.

### Unlock philosophy

Levels should unlock understanding, motivation, or convenience.

Good unlocks include:

* New collection views.
* Access to deeper challenge types.
* Social features.
* Cosmetic status markers.

Avoid unlocks that make early players feel excluded from the core loop.

### Coin sinks

Coins need meaningful uses or the economy inflates.

Priority sinks:

* Garage expansion.
* Cosmetic customization.
* Event participation or refreshes.
* Collection support tools.

Coin sinks should create choice. They should not create constant anxiety.

### Economy guardrails

* One activity should not dominate the economy.
* Duplicate farming should not outperform exploration.
* Passive reward collection should not exceed active play.
* Premium spending should not replace core progression.
* Late-game players should not break early-game balance.

### Inflation and pacing risks

Watch for four common failures.

#### Too little reward

Players feel stalled. Daily return value drops fast.

#### Too much reward

Levels lose meaning. Coin sinks stop mattering.

#### Reward concentration

One feature becomes optimal. Other systems feel irrelevant.

#### Reward opacity

Players stop understanding cause and effect. Progress starts to feel random.

### Player-facing feedback

* Show XP gained after each successful result.
* Show coin gains with a clear source.
* Preview upcoming unlocks where possible.
* Highlight first discovery bonuses separately.
* Distinguish base rewards from bonus rewards.
* Show duplicate conversion value clearly.

### Cross-system dependencies

This spec must align with several other systems.

* Rarity changes reward intensity.
* Daily challenges add temporary reward spikes.
* Collections add milestone payouts.
* Trading must not become the best progression shortcut.
* Monetization must respect normal earning expectations.

### Balancing questions

Use these questions during review.

* Does one normal session create visible progress?
* Do first discoveries still feel clearly better than duplicates?
* Can new players understand what coins are for?
* Do advanced players still have reasons to care about XP?
* Does daily play help progression without becoming required?

### Success criteria

* Players always know why they earned rewards.
* Progress feels steady across short sessions.
* High-value moments remain memorable.
* New players reach early unlocks quickly.
* Mid-game players see clear reasons to return.
* The economy supports retention without feeling manipulative.

### Out of scope

* Real-money pricing.
* Technical balancing formulas.
