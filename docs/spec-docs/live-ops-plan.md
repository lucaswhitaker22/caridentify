# Live ops plan

### Purpose

Keep the product active after launch. Create a sustainable rhythm for content and engagement.

### Product goals

* Give players regular reasons to return.
* Refresh collection goals without redesigning the core loop.
* Support retention and revenue over time.

### Live ops principles

* Events should extend the core loop, not replace it.
* New content should feel understandable immediately.
* The schedule must stay realistic for the team.

### Content layers

#### Daily layer

Use daily spotlight challenges and streak rewards. This keeps the product feeling active every day.

#### Weekly layer

Use rotating set goals, leaderboard resets, or featured collections. This creates short planning cycles for engaged players.

#### Seasonal layer

Use themed events, limited cosmetics, and exclusive rewards. This creates larger return moments across the year.

### Event design rules

* Every event needs one clear theme.
* Every event needs a visible start and end.
* Rewards should match the effort required.
* Event goals should support both casual and engaged players.

### Content planning guidance

* Start with simple recurring formats.
* Reuse proven mechanics before inventing new ones.
* Limit the number of overlapping live activities.

### Player communication needs

* Show what is live now.
* Show what ends soon.
* Explain event rewards clearly.

### Operational guardrails

* Do not rely on constant novelty.
* Do not overpromise cadence early.
* Do not make every event feel mandatory.

### Success criteria

* Players understand the current event at a glance.
* Events improve retention without causing fatigue.
* The team can maintain the schedule consistently.

### Out of scope

* Publishing workflows.
* Technical event configuration.
