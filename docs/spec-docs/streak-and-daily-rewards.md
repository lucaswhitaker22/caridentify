# Streak and daily rewards

### Purpose

Reward consistent return behavior. Build a lightweight daily habit.

### Product goals

* Increase returning users.
* Make daily opens feel worthwhile.
* Support retention without punishing breaks too hard.

### Core concept

Players earn escalating rewards for consecutive active days. The streak resets after a missed day.

### Reward rules

* Each consecutive day increases reward value.
* Rewards should cap at a defined point.
* Reaching streak milestones can unlock bonus rewards.
* Daily rewards should be claimable with minimal friction.

### Design principles

* Early streak rewards should feel immediate.
* Longer streaks should feel prestigious.
* Recovery from a reset should not feel impossible.

### UX principles

* Show current streak clearly.
* Preview tomorrow's reward.
* Celebrate milestone streaks.

### Guardrails

* Do not make streak loss feel devastating.
* Do not tie essential progress only to streaks.
* Do not create pressure that harms healthy play patterns.

### Success criteria

* Players understand the streak system instantly.
* Daily opens increase after launch.
* The system supports retention without burnout.

### Out of scope

* Notification scheduling.
* Technical timestamp validation.
