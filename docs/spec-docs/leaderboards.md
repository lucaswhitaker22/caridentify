# Leaderboards

### Purpose

Create lightweight competition. Let players compare progress without direct combat.

### Product goals

* Add social motivation.
* Reward consistent play.
* Highlight different kinds of achievement.

### Ranking categories

* Total cars collected.
* Rare cars collected.
* Total XP.
* Daily streak length.

### Design rules

* Rankings should update frequently enough to feel live.
* Players should see their own position clearly.
* Top ranks should feel aspirational, not unreachable.
* Leaderboards should support both global and friend scopes.

### Competitive principles

* Reward effort across multiple play styles.
* Avoid one ranking that defines the whole game.
* Show movement over time when possible.

### Fairness guardrails

* Suspicious accounts should not dominate rankings.
* New players need a realistic path to visible progress.
* Friend leaderboards should stay easy to access.

### UX principles

* Keep each leaderboard easy to read.
* Show the metric behind the rank.
* Surface notable milestones and rank changes.

### Success criteria

* Players check leaderboards regularly.
* Rankings increase repeat sessions.
* Leaderboards support status without creating frustration.

### Out of scope

* Anti-abuse enforcement details.
* Backend sync behavior.
