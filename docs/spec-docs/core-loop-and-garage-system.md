# Core loop and garage system

### Purpose

Define the main player loop. Set the standard for speed, clarity, and reward.

### Product goals

* Make the first successful capture feel instant.
* Keep each session simple to start.
* Turn every new car into collection progress.

### Player promise

A player can spot a car, capture it, identify it, and save it fast. The flow should usually complete in under 10 seconds.

### Primary flow

1. Player opens the camera.
2. Player captures one vehicle.
3. The app identifies the car.
4. The app shows the result card.
5. If the car is new, it enters the garage.
6. The player earns rewards.

### Required outcomes

* Every successful scan returns one clear car identity.
* Every new car creates a collectible card.
* Every duplicate still gives a useful outcome.
* The garage always reflects the latest collection state.

### Garage rules

* The garage is the player's permanent collection.
* Each entry shows make, model, generation, and rarity.
* New entries feel distinct from duplicates.
* The garage supports browsing by set and rarity.

### Duplicate handling

* A duplicate never blocks progress.
* A duplicate grants coins, XP, or trade value.
* A duplicate should feel less valuable than a first discovery.

### UX principles

* Prioritize one obvious action per screen.
* Keep the capture result easy to scan.
* Celebrate first-time finds.
* Reduce friction between scans.

### Success criteria

* Players understand the loop without a tutorial.
* First-time users complete a successful scan quickly.
* Returning users can chain multiple scans in one session.

### Out of scope

* Manual car entry.
* Multi-car capture in one flow.
* Deep technical identification details.
