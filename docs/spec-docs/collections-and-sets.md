# Collections and sets

### Purpose

Group cars into meaningful themes. Turn isolated captures into long-term collection goals.

### Product goals

* Extend retention through completion goals.
* Make the garage feel curated.
* Reward exploration across car categories.
* Create progress that stays valuable beyond raw count.

### Player value

Sets answer a simple motivation. "What should I try to complete next?"

They give players structure without forcing a single path. They also make common, rare, and duplicate outcomes feel connected to a bigger goal.

### Core concept

A set is a themed group of cars. Completing a set grants a trophy, status, or bonus reward.

Sets should create medium-term and long-term goals. A player should usually have one set close to progress and one set worth chasing.

### Example set themes

* Japanese sports cars from the 1990s.
* American muscle icons.
* Electric pioneers.

### Why sets matter

The core loop creates single moments. Sets turn those moments into sustained intent.

They help the garage feel organized. They also create reasons to care about specific cars, not only overall volume.

### Set design rules

* Every set needs a clear theme.
* A set should feel achievable over time.
* Sets should vary in size and rarity mix.
* Set progress should update automatically.
* The title and theme should be understandable instantly.
* Most sets should combine attainable progress with one or two chase targets.

### Set types

Use a mix of set shapes to support different motivations.

#### Intro sets

Small, readable, and friendly to new players. These teach how set completion works.

#### Standard sets

The main format for regular play. These support steady collection progress across weeks.

#### Prestige sets

Harder to complete and more status-driven. These should be limited in number and clearly framed as aspirational.

#### Event sets

Time-bound sets tied to seasonal or live content. These should reuse familiar rules and stay clearly labeled.

### Composition guidance

A good set has internal variety.

* Different makes or subtypes can support the theme.
* Rarity should add tension without blocking progress.
* Set size should match expected player effort.

Avoid sets that feel random. Avoid sets that are so broad that the theme loses meaning.

### Reward principles

* Partial progress should feel visible.
* Full completion should grant a memorable reward.
* High-difficulty sets should offer prestige value.
* Rewards should match set difficulty and rarity mix.
* Set rewards should not outscale the main progression loop.

### Progress model

Players should always know three things.

* What the set theme is.
* How many items they already have.
* Which missing items matter most.

Near-complete sets should surface naturally. This helps players choose a next target without extra friction.

### Reward structure

Use layered rewards instead of one flat payout.

#### Partial completion

Progress itself should feel satisfying. This can come through visible percentages, milestone markers, or small thresholds.

#### Full completion

Completion should give a stronger reward spike. Good outcomes include coins, XP, badges, cosmetics, or trophy-style recognition.

#### Prestige value

Hard sets should carry status beyond raw currency. Recognition often matters more than payout here.

### Discovery experience

* Players can browse available sets.
* Each car card can show set membership.
* The garage can highlight near-complete sets.
* Missing entries should feel motivating, not confusing.

### UX expectations

Set browsing should answer intent quickly.

* Show set theme and completion state first.
* Let players inspect missing items easily.
* Make membership visible on result and garage surfaces.
* Highlight when a new capture advances multiple sets.

### Cross-system dependencies

Sets depend on several surrounding systems.

* Rarity shapes chase value.
* Progression defines reward intensity.
* Trading helps players close gaps.
* Achievements can celebrate set milestones.
* Live ops can introduce time-bound set themes.

### Guardrails

* Do not overload new players with too many sets.
* Do not hide required cars behind paywalls.
* Do not make only rare cars matter.
* Do not flood the garage with overlapping low-value sets.
* Do not make event sets feel mandatory for core progression.

### Failure risks

Watch for three common problems.

#### Theme weakness

Sets feel arbitrary. Players stop caring about completion.

#### Completion fatigue

Too many long sets compete at once. Progress loses emotional impact.

#### Reward mismatch

Hard sets pay too little or easy sets pay too much. Motivation becomes distorted.

### Review questions

Use these questions when evaluating a new set.

* Is the theme immediately understandable?
* Does the set include both steady progress and chase moments?
* Would a casual player still feel progress here?
* Does the completion reward fit the effort required?

### Success criteria

* Players understand how set completion works.
* Set progress increases repeat play.
* Completion rewards feel worth the effort.
* Players can identify a next target without confusion.
* Sets make the garage feel more purposeful over time.

### Out of scope

* Content database design.
* Technical completion checks.
