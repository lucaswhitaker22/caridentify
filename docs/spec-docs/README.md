# Spec Docs

### Purpose

Define the full product design in one place. Show how each system supports capture, collection, retention, and fairness.

### How to use these specs

Start with the [Overview](../). Then read the core systems first.

Use the later specs to add depth. Each page should answer player value, rules, guardrails, and success criteria.

### Product design principles

* The first successful capture must feel fast.
* Progress must stay visible in every session.
* Rare moments must feel exciting, but still fair.
* Social systems should add value, not pressure.
* Live ops should refresh goals without creating confusion.

### System relationships

The product stands on four layers.

1. **Foundation.** The core loop, garage, progression, and rarity define the base experience.
2. **Engagement.** Challenges, sets, quizzes, and streaks create repeat reasons to return.
3. **Social.** Leaderboards, badges, and trading turn progress into recognition.
4. **Operations.** Fair play, monetization, roadmap, personas, journey mapping, and live ops keep the product sustainable.

### Cross-spec guardrails

Use these rules when two specs compete.

* Protect the first-time capture experience first.
* Prefer clarity over depth in early releases.
* Make duplicates useful before adding rarity complexity.
* Add social pressure only after solo value feels strong.
* Never let monetization bypass core progression.
