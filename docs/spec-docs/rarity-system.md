# Rarity system

### Purpose

Use rarity to create excitement, hierarchy, and long-term chase value. Make important finds instantly recognizable.

### Player value

Rarity turns a normal result into a spectrum of excitement.

It gives collectors status targets. It gives the reward system stronger peaks. It gives live events flexible design space.

### Rarity tiers

* Common
* Uncommon
* Rare
* Epic
* Legendary

### Product goals

* Give every capture a visible status.
* Create aspirational goals for collectors.
* Support rewards, sets, and event design.
* Keep rarity meaningful without making common finds worthless.

### What rarity should mean

Rarity should signal importance, not only scarcity.

A higher tier can reflect real-world rarity, cultural prestige, historical interest, or event importance. The meaning must stay intuitive.

### Player experience

Rarity appears on every result card. Players should understand it at a glance.

A rare result should feel special immediately. The reaction should come from presentation, rewards, and collection impact together.

### Design rules

* Rarity should reflect real-world scarcity or significance.
* Rarity must stay stable once assigned.
* Higher rarity should increase prestige and rewards.
* Rarity should remain understandable at a glance.
* Tier boundaries should not feel arbitrary.
* Event exceptions should stay rare and clearly labeled.

### Tier intent

#### Common

Forms the foundation of the collection. Supports early wins and set completion.

#### Uncommon

Adds light variety. Starts teaching that some finds matter more.

#### Rare

Introduces true chase value. Should feel exciting, but still attainable.

#### Epic

Represents high-prestige finds. Should create memorable result moments.

#### Legendary

Represents the strongest prestige tier. Should stay unusual in standard play.

### Assignment guidance

Rarity should be assigned deliberately. It should not be evenly distributed.

Use these inputs:

* Real-world visibility.
* Collector appeal.
* Cultural recognition.
* Product pacing needs.

Do not assign rarity only to force monetization or event urgency.

### Usage across the product

* Rewards scale by rarity.
* Challenges can feature rarity targets.
* Collections can require mixed rarity goals.
* Leaderboards can track rare finds separately.
* Achievements can celebrate rarity milestones.
* Trading can assign stronger social value to high-tier duplicates.

### Reward implications

Rarity should affect both emotion and economics.

Higher rarity can increase:

* XP rewards.
* Coin rewards.
* Collection prestige.
* Social share value.

Higher rarity should not guarantee overall player dominance. Consistency still matters.

### Collection design implications

Common and uncommon cars make collections feasible. Rare and above create chase goals.

A good set mixes attainable progress with standout targets. A bad set blocks completion behind a single extreme tier.

### Event usage

Rarity can shape event goals in controlled ways.

Examples:

* Spotlight a rarity band for bonus rewards.
* Run a themed week around rare finds.
* Add limited cosmetic recognition for difficult rarity goals.

Event use should amplify normal rarity meaning. It should not rewrite it constantly.

### Guardrails

* Legendary cars must stay uncommon in normal play.
* Common cars must remain valuable for set completion.
* Rarity alone should not define player success.
* Rare tiers must not become paywalled progress.
* Event boosts must not destroy long-term rarity trust.
* Trading must not make top rarity trivial to acquire.

### Fairness risks

Watch for three failure modes.

#### Over-distribution

Rare tiers appear too often. Excitement collapses.

#### Under-distribution

Players stop believing high tiers are realistic. Motivation collapses.

#### Meaning drift

Tier labels stop matching player intuition. Trust declines.

### Presentation guidelines

* Use distinct labels and visual treatment.
* Keep labels short and familiar.
* Never hide rarity behind extra taps.
* Make high-tier reveals feel stronger, not slower.
* Keep the visual ladder consistent across all surfaces.

### Cross-system dependencies

Rarity touches almost every engagement system.

* Progression uses it for reward scaling.
* Daily challenges use it for rotating goals.
* Sets use it for collection pacing.
* Trading uses it for perceived value.
* Leaderboards use it for prestige comparisons.

### Success criteria

* Players understand rarity after one session.
* Rare findings generate excitement and sharing behavior.
* Rarity improves retention without causing frustration.
* Common finds still support meaningful progress.
* High-tier moments remain memorable over time.

### Out of scope

* Dataset rules.
* Technical classification logic.
