# Daily spotlight challenges

### Purpose

Create a reliable daily return hook. Turn open-ended exploration into a focused short-term goal.

### Product goals

* Increase daily opens.
* Encourage players to explore beyond habit.
* Create shareable wins.
* Add urgency without punishing missed days.

### Core concept

Each day features one highlighted car target. Spotting that target grants bonus rewards and recognition.

The spotlight gives the day a clear objective. It should answer "What should I chase today?" immediately.

### Player value

Daily challenges reduce decision fatigue. They give casual players direction and engaged players routine.

They also create lightweight community moments. Players can compare whether they hit the same target.

### Challenge rules

* One spotlight challenge runs each day.
* The target is visible before the player captures.
* Completion grants bonus XP, coins, and a badge or event marker.
* A completed challenge cannot be repeated for full value on the same day.
* Daily reset should happen at a predictable global time.
* The challenge should require normal play, not unusual setup.

### Loop design

The loop should stay simple.

1. Player sees today's target.
2. Player understands the reward.
3. Player goes exploring with a clear intention.
4. Player completes the challenge or expires at reset.
5. The product previews the next return reason.

### Target selection rules

Target choice shapes the whole experience.

Use these principles:

* Most targets should be realistically achievable.
* Some targets can stretch ambition during events.
* Targets should vary by make, model, region, or rarity.
* Consecutive days should not feel repetitive.

Avoid targets that depend on luck alone. Avoid targets that exclude newer players too often.

### Reward design

* Rewards should be better than a normal scan.
* The bonus should feel worth chasing.
* Completion should be celebrated with a clear result state.
* Rewards should support, not replace, the main economy.

#### Recommended reward mix

Use a mix of immediate and cumulative value.

* Bonus XP for visible account progress.
* Bonus coins for flexible utility.
* A marker, stamp, or badge for history and status.

This makes completion feel good today and still matter later.

### Difficulty guidelines

* Most daily targets should be achievable.
* Some targets can be aspirational during events.
* Challenge difficulty should vary across the week.

#### Difficulty model

Use a predictable spread.

* Easy days build confidence.
* Standard days support the main habit.
* Stretch days create excitement and conversation.

Do not stack too many stretch days together. That turns urgency into fatigue.

### Failure and rollover rules

* Missing a day should not punish long-term progress.
* Unfinished challenges expire at reset.
* New players should receive accessible early targets.

### Anti-frustration guardrails

* Do not tie streak loss to missing a spotlight.
* Do not require premium spend for completion.
* Do not hide the target behind prior wins.
* Do not make failure feel like wasted play.

Even failed attempts can still grant normal scan rewards. The spotlight bonus is extra, not all-or-nothing value.

### Event variants

The spotlight system should scale during events.

Possible variants:

* Themed weekly spotlights.
* Rarity-focused spotlight days.
* Community challenge weekends.

Variants should reuse the same mental model. Players should not need new rules every week.

### UX principles

* Show the daily target on the home screen.
* Remind players of time remaining.
* Make completion unmistakable.
* Surface progress without adding clutter.
* Explain why the target is interesting.

### Cross-system dependencies

This system depends on several others.

* Progression defines the reward value.
* Rarity affects difficulty and excitement.
* Live ops determines cadence and themes.
* Streaks must complement the system, not compete with it.

### Review questions

Use these questions when tuning the feature.

* Does the target feel achievable today?
* Is the reward clearly better than a normal scan?
* Would missing today feel disappointing, but not punishing?
* Does this challenge encourage different exploration behavior?

### Success criteria

* Players return to check the daily target.
* Challenge completion improves retention.
* Daily rewards feel motivating, not required.
* The system stays understandable after one use.
* Daily participation supports the main loop instead of replacing it.

### Out of scope

* Technical validation methods.
* Live event scheduling systems.
